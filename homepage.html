Stackage is a stable source of Haskell packages. We guarantee that packages build consistently and pass tests before generating nightly and Long Term Support (LTS) releases.

Why

A tested package set is the best solution to dependency resolution issues (a.k.a. Cabal Hell). By using Stackage, you’ll avoid many common build problems, have a stable package set for personal work, and a common environment for team collaboration.

Quick start

  1. Download and install stack.
  2. Run stack build
  3. Done!

Using cabal

Download the following file, place it in your project's root directory (next to your project's .cabal file) and you’re done. 

$ wget https://www.stackage.org/lts/cabal.config
$ cabal update
$ cabal install

NOTE: Requires GHC 7.8. For GHC 7.10, please change the URL to https://www.stackage.org/nightly/cabal.config

Get Started

  • LTS Haskell (what is Long Term Support Haskell?)
  • Stackage Nightly

You’ll need to install Haskell first. LTS Haskell requires GHC 7.8, and Stackage Nightly requires GHC 7.10. Then use all your standard Haskell tooling features, like cabal and sandboxes.

You may also be interested in always getting the latest LTS or Nightly release.

All packages included

Stackage includes all packages by default. However, we can only guarantee buildability for officially supported packages. To get a package added to our officially supported list, please follow the author/maintainer instructions.
